# Publio 编辑工作台 UI 设计稿

## 背景

Publio 当前已经有首页编辑器、AI 新闻页和设置页，但整体界面存在明显割裂：

- 首页偏暖色内容工具风格
- 预览区使用深色视觉孤岛
- AI 新闻页像普通资讯页
- 侧边栏与卡片层级更接近常规后台模板

这些页面分别能工作，但没有形成统一的产品气质，因此整体观感不够专业，也难以支撑“选题判断 + 写作生产 + 多平台分发”这一整条工作流。

本次 UI 设计的目标不是做局部美化，而是为产品建立一套统一、可持续扩展的界面语言。

## 目标

- 为 Publio 建立统一的视觉与交互系统
- 让产品更像“编辑决策工作台”，而不是多个页面的拼接
- 提升选题、研究、写作、发布四个阶段的界面连续性
- 强化专业感、信息层级与内容判断氛围
- 为后续使用 `frontend-design` 实施重设计提供明确边界

## 非目标

- 本次设计不追求品牌营销站式的强视觉表达
- 不追求娱乐化、社交化或创作者玩具式界面
- 不追求复杂三维拟物、沉浸式大动效或重插画风格
- 不重新定义产品信息架构之外的业务能力

## 视觉定位

### 核心方向

本次 UI 方向定义为：

> 以“中文商业媒体编辑部”为主气质，借用“现代研究终端”的信息组织方式，最终形成一套浅色纸面感的商业媒体情报桌。

### 气质关键词

- 浅色纸面
- 锋利分栏
- 商业媒体感
- 研究终端秩序
- 强信息层级
- 专业但不冷漠

### 不采用的方向

以下方向明确不采用：

- 奶油暖色内容工具风
- 通用 SaaS 模板风
- 炫技渐变创作者工具风
- 深色情报屏 / 监控大盘风

## 用户感知目标

用户打开产品后的第一感受应该是：

- 这是一套给专业内容人使用的工作台
- 信息很多，但被整理得很清楚
- 这里不仅能写，还能帮助判断什么值得写
- 每个页面都像同一套系统的一部分

产品最终应给人留下的记忆点不是某个装饰颜色，而是：

> 像一张信息高度整理过的商业媒体情报桌。

## 一、整体界面语言

### 1. 材质基调

整体采用浅色纸面感，但不能是温吞的奶油白。更准确的材质感应接近：

- 高质量纸张
- 编辑部桌面文件
- 商业研究报告页

背景建议使用略带矿物感的浅灰米色，而不是纯白。这样既能保留内容产品的可读性，又能形成稳定的专业气质。

### 2. 分栏感

界面的结构感应该比当前更强。不是靠重阴影、重描边制造“组件感”，而是靠清晰的栏位、区块和内容密度建立秩序。

页面布局要像桌面工作区，而不是居中堆卡片的营销页面。

### 3. 信息优先

视觉语言应服务内容判断，因此：

- 标题必须有存在感
- 结构信息必须易扫描
- 注释信息要低调但清楚
- 强调色只用于关键信号和主操作

## 二、页面骨架设计

整个产品保持统一的工作台骨架，但不同页面根据任务稍作调整。

### 1. 全局框架

统一采用：

- 左侧：产品导航
- 中间：当前主任务区
- 右侧：上下文 / 研究 / 预览 / 状态区

不是所有页面都必须机械三栏，但都应共享同一节奏。

### 2. 选题台

选题台是整个产品的第一工作区，不应再被设计成普通新闻列表页。

结构建议为：

- 左侧：全局导航
- 中间：候选池列表
- 右侧：研究底稿与编辑判断

中间列表分为两个主分组：

- `今天能发`
- `还能追`

每个候选卡片展示的不是长摘要，而是：

- 标题
- 为什么值得写
- 影响对象
- 来源密度
- 推荐切口

右侧详情区保持稳定版式，不随内容切换而大幅跳动，让用户形成“左侧筛、右侧判”的工作记忆。

### 3. 写作台

写作台不应只是裸露的 Markdown 编辑器，而应成为“围绕某个选题进行创作”的工作区。

结构建议为：

- 顶部：稿件标题与选题来源
- 左侧主区：正文编辑
- 右侧：发布预览
- 上方或右上附属区：研究上下文
- 底部或下方：平台分发与状态反馈

写作台进入时应让用户明确知道：

- 当前在写什么
- 这篇稿来自哪个候选题
- 为什么这题值得写
- 当前建议切口是什么

### 4. 设置与平台

设置页不需要承担风格探索任务，应保持最克制、最稳定的结构，用于：

- 平台授权
- 环境配置
- 发布参数维护

它应延续工作台视觉语言，但信息密度低于选题台和写作台。

## 三、视觉系统

### 1. 色彩系统

#### 主背景

- 使用浅纸白、矿物灰、微米色作为基础背景层
- 避免使用纯白大底与高饱和暖色大面积覆盖

#### 主文本

- 使用偏墨黑或深棕黑，而不是纯黑
- 强调阅读舒适度与长时间使用稳定性

#### 强调色

- 使用单一锈红 / 铜橙作为主强调色
- 用于主按钮、当前激活、关键标签、重要判断提示
- 不让强调色承担大面积背景角色

#### 冷色辅助

- 可少量使用冷灰蓝或钢灰蓝承载结构性信息
- 用于时间、来源、评分、辅助标签等
- 只承担“秩序信息”，不承担品牌主视觉

### 2. 排版系统

排版不应该只是“字号大小变化”，而要体现编辑与研究的双重气质。

#### 标题层

- 页面主标题应具备商业媒体感
- 可使用有节制的 serif 或 semi-serif 作为标题字体
- 保持锋利、成熟、有判断感

#### 正文层

- 使用清晰克制的 sans-serif
- 服务于说明、研究底稿、表单和正文阅读

#### 辅助信息层

- 时间、来源、分组标题、状态标签要更紧凑
- 更像研究报告里的注释，而不是营销界面的 badge

### 3. 卡片与面板系统

所有卡片与面板应像同一套文档家族。

统一原则：

- 边框清晰但不厚重
- 阴影浅、短、干净
- 圆角不宜过软，略带锋利感
- 内部排版稳定，减少视觉花活

候选卡片、研究面板、编辑器容器、发布状态面板应具有家族一致性。

### 4. 按钮与控件系统

主按钮不应过胖、过亮、过消费产品化。更适合采用：

- 克制的填充按钮
- 明确但不过度炫耀的主色
- 辅助操作使用文字按钮、线框按钮或工具按钮

控件语言整体要更接近“编辑动作”，而不是“营销转化动作”。

## 四、交互原则

### 1. 主次明确

每个页面必须让用户一眼知道：

- 主任务是什么
- 当前焦点在哪里
- 下一步最合理的动作是什么

### 2. 切换稳定

选题切换、底稿切换、预览切换都应尽量保持版式稳定，避免页面大范围重排。

### 3. 动效克制

只保留少量高质量动效：

- 页面进入时的分层显现
- 候选切换时右侧内容平滑更新
- hover 时的信息点亮与边框变化

不使用：

- 大面积浮夸位移动画
- 过度缩放
- 与工作场景无关的炫技效果

### 4. 信息点亮而非组件表演

交互反馈应该让用户感觉“信息被激活了”，而不是“组件在卖萌”。这对专业感非常关键。

## 五、页面级设计建议

### 1. 选题台页面

页面关键词：

- 档案式候选卡片
- 固定研究面板
- 双层候选池
- 信息筛选优先

核心体验：

- 扫描候选
- 快速判断
- 查看依据
- 加入写作台

### 2. 写作台页面

页面关键词：

- 带研究上下文的编辑器
- 左写右看
- 分发区与正文区一体化
- 像真实工作桌，而不是工具演示页

核心体验：

- 带着判断开始写
- 写的过程中持续参考上下文
- 预览与分发自然衔接

### 3. 导航与全局框架

导航不应只是菜单，而应体现产品角色。建议文案与排序更贴近真实工作流：

- `选题台`
- `写作台`
- `设置与平台`

这样用户一进来就能理解整个产品的使用顺序。

## 六、设计 token 建议

后续实现时应抽出统一 token，而不是继续依赖每页内联拼色。

建议至少抽出：

- 页面背景色
- 面板背景色
- 主文本色
- 次文本色
- 边框色
- 强调色
- 强调色弱背景
- 阴影层级
- 页面宽度与间距
- 卡片圆角与面板圆角

这些 token 应优先落在全局样式中，为 `frontend-design` 实施提供一致基础。

## 七、与功能设计的一致性

本次 UI 设计必须服务前面确认的产品方向：

- AI 新闻不再是资讯流，而是候选题编辑台
- 输出重点不是自动成稿，而是研究底稿
- 主流程是“选题 -> 判断 -> 写作 -> 发布”

因此 UI 必须优先体现：

- 判断
- 取舍
- 结构化信息
- 连续工作流

而不是单纯做得“更漂亮”。

## 八、实施优先级建议

后续使用 `frontend-design` 落地时，建议按以下顺序推进：

### 阶段 1：全局视觉基线

- 重写全局背景、色彩 token、排版和基础容器
- 统一侧边栏与页面头部

### 阶段 2：选题台重设计

- 先把最能体现新产品价值的页面做出来
- 建立候选卡片与研究面板的家族语言

### 阶段 3：写作台重设计

- 把编辑器、预览、研究上下文和分发区统一进同一视觉系统

### 阶段 4：设置与平台收尾

- 把低频页面收进统一体系，保证完整度

## 设计结论

Publio 的 UI 重设计不应被理解为“换一套更好看的皮肤”，而应被理解为：

> 把一个已有功能的多页面工具，重构成一套真正有工作流逻辑、有编辑判断气质、有研究秩序感的专业内容工作台。

最适合它的视觉答案不是温暖创作者风，也不是冷硬监控台，而是：

> 一张浅色纸面感的商业媒体情报桌。
